想定ユーザーと利用シーン

仙台エリア担当 営業5名 / iPhone・iPad 屋外使用

飲食店・給食・宿泊・小売など幅広い取引先を、一人あたり週20〜40件規模で担当。車内・路上・店内での片手操作と日光下の可読性が最重要。

PRIMARY USER
営業担当（5名）
山田・鈴木ら現場営業。運転の合間・店舗訪問直後に片手で使う前提。文字入力する余裕はない。
MANAGER
営業マネージャー
訪問状況を俯瞰。停滞 Account の早期発見と次アクション追跡を Web 画面で。
ADMIN
情シス / 営業事務
AppSheet Editor で View・式を微調整できる。非エンジニアでも改修可能な構造が運用負担を下げる。

3-3 · ACCOUNT DETAIL

取引先詳細（焼肉 大吉）

停滞Accountの赤ピンをタップして開く画面。0.5秒で欲しい情報を上から優先順で配置：会社名 → ランク → 最終訪問経過日 → 進行中商談 → 主要アクション。

  • 「最終訪問: 162日前」を赤で強調し、危機感を色で伝達
  • 訪問を開始 を画面上部1/3 以内に配置、迷わずタップ
  • 訪問履歴はDeckで時系列、4件目以降は別画面へ退避
3-4 · VISIT RECORD QUICK
★最重要

訪問メモ（音声クイック入力）

本プロダクトの体験の核。現場の営業が3秒で訪問を記録できることが「使い続けられるか」の分水嶺。録音ボタンは画面最上部・幅100%・80px高。

  • 3秒体験: 録音だけして閉じてもOK（他は任意）
  • 10秒体験: 録音 + 会った人1タップ + 用件1タップ
  • 30秒体験: 上記 + 次アクション1行
  • 画面下部固定の「✓ 訪問を終了」で1タップ完了

3-5 · TODAY'S ROUTE

今日のルート（Deck）

今日の訪問予定を時系列カードで一覧。番号・時刻・状態バッジの3点セットで、運転中にチラ見で進捗が読める。ナビ起動は1タップで Google Maps へ。

  • 訪問済は取消線と滞在時間、今日の成果も同画面で把握
  • 画面下に移動距離・所要時間サマリ
  • Phase 2 で順序最適化（現状は scheduled_at 順）
3-6 · VISIT DETAIL

訪問詳細（和食 瑞鳳 / 04-15）

完了済み訪問のレコード。後から振り返る用途のため、メモ本文を全文表示して滞在時間・用件・次アクションの期限まで一画面で把握可能。

  • 滞在時間 50分 を自動計算
  • 抽出された「次アクション / 期限」を別カードで表示
  • 商談化した note にはOpportunity化済バッジ

FEATURES

機能ハイライト

Phase 1 で確実に価値を出す4機能と、Phase 2 で追加する2機能の計6本柱。

地図で取引先を俯瞰

ランク × 最終訪問日の2軸でピン色分け。停滞Accountが赤で一目瞭然。

3秒で商談を残す音声メモ

画面最上部の大型マイクボタン。録音して閉じるだけで記録完了。

取引先ごとの商談タイムライン

訪問履歴・商談・担当者が Account Detail で時系列に並ぶ。

停滞Accountの自動ハイライト

Aランク × 60日未訪問を毎朝自動検出、Home のバッジと朝7:30のアラート通知。

Phase 2

Phase 2

Gemini Speech で音声→テキスト、Gemini Vision で名刺OCR。次アクション抽出まで自動化。

TECH STACK

技術スタックと構築方針

コードを書かないノーコード構築。保守は非エンジニアでも可能。

APP PLATFORM
AppSheet (Google)

Google 公式ノーコード。ビュー・アクション・自動化を設定ベースで構築。

DATABASE
Supabase PostgreSQL

本番は CRM と共用。デモは Google Sheets でも可（サンプル10件）。

MAP
Google Maps Platform

地図描画・住所→座標・ルート最適化・ナビ起動を一括提供。

STORAGE
Google Drive

音声・画像ファイルは Workspace 容量内に保存。別途ストレージ契約不要。

AUTOMATION
Apps Script

ルート最適化・音声文字起こし・通知など AppSheet の外側を担当。

AI
Gemini (Google)

音声文字起こし / 次アクション抽出 / 名刺 OCR。Phase 2 以降。

APPROACH 1
コードを書かないノーコード構築

AppSheet Editor 上で View・Action・Bot を GUI 設定。スキーマ変更と式編集のみで完結します。

APPROACH 2
保守は非エンジニアでも可能

「色を変えたい」「フィルタ追加したい」は情シス・営業事務が直接調整できる構造。開発依存から脱却します。
