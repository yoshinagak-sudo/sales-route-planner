# 管理者ダッシュボード（/admin → ダッシュボードタブ）

最終更新: 2026-04-24

## 目的
営業マネージャー・管理者が「全営業の活動状況」を 1 画面で把握できるビュー。
既存 `/admin` の Tabs に「ダッシュボード」を先頭追加し、`defaultValue` を変更する。

## 画面構成（縦並び）

```
┌─ ヘッダー（既存）「マスタ管理」───────────────────────────┐
└─────────────────────────────────────────────────────────────┘
┌─ Tabs ──────────────────────────────────────────────────────┐
│ [ダッシュボード] [取引先(N)] [連絡先(N)] [ユーザー(N)]       │
└─────────────────────────────────────────────────────────────┘

  TabsContent: ダッシュボード
  ├── 1) KPI サマリー (4 カード横並び)
  │     [本日の訪問予定]  [未記録の訪問]  [停滞 A ランク]  [進行中の商談]
  │
  ├── 2) 営業ごとの活動表
  │     営業 / 部署 / 担当数 / 今日の予定 / 完了 / 未記録 / 停滞A
  │
  ├── 3) 停滞アカウント横断一覧 (最大10件、最終訪問が古い順)
  │     会社名 / ランク / 担当営業 / 最終訪問日 / 経過日数(赤強調)
  │     行クリック → /accounts/[id]
  │
  ├── 4) 進行中商談一覧 (status=OPEN, 金額降順)
  │     取引先 / 案件名 / 金額 / 期日 / 担当 / 次のアクション
  │     期日超過行は text-danger 強調
  │
  └── 5) 過去7日間の活動量
        - 訪問件数 / 平均所要時間 / 担当別件数 (横並びテキスト)
```

## レスポンシブ
- KPI: モバイル 2 列 (`grid-cols-2`) → md 以降 4 列 (`md:grid-cols-4`)
- 表は overflow-x-auto でスクロール可能。担当者表のみコンパクトに収める
- 停滞リストはテーブル。モバイルでは右側カラム（経過日数）を優先表示

## デザイントークン
- KPI カード: 既存 SummaryCard 相当のパターン (border + tone) を踏襲
- 期日超過 / 停滞: `text-danger`, `bg-danger/5`
- 停滞ランク: `BadgeRank` 既存
- アクセントが必要な箇所のみ `bg-accent-amber/10`
- 表ヘッダ: `bg-muted/40`

## アクセシビリティ
- 各セクションに `<section aria-labelledby>` で見出しを紐づけ
- 数値は `tabular-nums` で桁揃え
- 期日超過は色だけでなく "超過" テキストでも判別可能に

## 実装メモ
- Server Component のまま。useState なし
- `mockUsers` / `mockAccounts` / `mockVisits` / `mockVisitNotes` / `mockOpportunities` を直接 import
- 今日判定は `getVisitsToday` を使わず Server 内で再計算（管理者は全営業横断のため）
- 停滞 A は `daysSince(a.lastVisitAt) >= 60 && a.rank === "A"`
- 未記録は `mockVisits` の COMPLETED で `mockVisitNotes` に visitId 紐付きが無いもの
- 7 日間活動量は `(now - leftAt or scheduledAt) <= 7 day && COMPLETED`
